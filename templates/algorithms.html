Implemented Algorithms

SHA-256 Hashing

Every uploaded certificate is converted into a SHA-256 digest. Verification recalculates the digest and detects tampering when hashes differ.

Shard Allocation

The shard coordinator hashes certificate metadata such as issuer, course and certificate ID. It assigns the record to one of three logical shards and applies load balancing when one shard becomes heavier.

Corda Notary Simulation

The application prevents duplicate active certificate files through hash consistency checks and records every issue/revoke transaction as a unique ledger block.

Metadata Versioning

Reissuing the same certificate ID creates a new version while retaining older version metadata for auditing.

Reason-Tagged Revocation

Revocation changes the certificate status, stores revocation time and reason, creates a revoke block, and updates verification output.

QR-Based Verification

Each certificate receives a QR code that opens the verification route for that certificate ID.

Off-Chain Storage Simulation

The certificate file is stored outside the ledger. The app stores only hash, CID-like reference and metadata in JSON ledger records.

Fraud Detection

The app flags duplicate certificate hashes and tampered files using hash comparison during issue and verification.
